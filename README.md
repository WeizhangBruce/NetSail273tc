# NetSail 2.73 TC（網路航海時代）

NetSail 2.73 TC 是一套以 Perl CGI 寫成的多人網頁航海遊戲。玩家扮演船長，建立艦隊後在世界各海域與港口之間航行，透過買賣交易品、雇用水手、購買與修理船隻、聽取冒險情報、尋寶、與其他玩家交戰或經營港町來累積資金、名聲與經驗。

本倉庫是「ネット航海時代」2.73 的繁體中文化／改造版本，主要程式仍維持傳統 CGI 架構：每次 HTTP 請求由 `.cgi` 腳本讀取表單參數、載入純文字資料檔、執行遊戲邏輯，再輸出 HTML。

## 專案結構

```text
.
├── sea.cgi              # 遊戲主入口：登入後主畫面、模式分派、狀態顯示、港內行動
├── new.cgi              # 玩家註冊與初始角色建立
├── admin.cgi            # 管理者功能：帳號管理、管理員建號、公告／訊息
├── setting.cgi          # 全域設定：資料目錄、CGI 路徑、UI、密碼／IP 限制等
├── sys.cgi              # 共用系統函式：玩家資料讀寫、Cookie、表單解碼、HTML 外框
├── Nfile.pm             # 檔案讀寫與鎖定模組，提供資料檔的互斥存取
├── move.cgi             # 航行目的地列表、移動時間、食物消耗與抵達後物價回歸
├── trade.cgi            # 港口交易品買賣、價格波動、玩家貨物更新
├── yard.cgi             # 造船所：購船、賣船、修船、能力提升品
├── bar.cgi              # 酒場：水手／食物管理、冒險情報、同港玩家互動
├── battle.cgi           # 玩家艦隊戰鬥、戰術、戰利品、沉船與戰鬥限制
├── city.cgi             # 港町主畫面與建城流程
├── csys.cgi             # 城鎮資料讀寫
├── ctrade.cgi           # 城鎮市場、銀行、購買城鎮庫存／船艦／財寶
├── cadmin.cgi           # 城鎮擁有者管理：資金、修復、上架貨物／船艦／財寶、出售城鎮
├── cbattle.cgi          # 攻擊港町與破壞工作
├── citem.cgi            # 城鎮財寶交易相關流程（部分功能與 ctrade/cadmin 重疊）
├── list.cgi             # 玩家列表、個人資料、履歷、事件列表等彈窗頁面
├── mail.cgi             # 玩家間訊息
├── rank.cgi             # 首頁排行榜
├── event.cgi            # 全服事件記錄寫入
├── intro.cgi            # 玩家自我介紹
├── chat.cgi             # 內嵌聊天室入口
├── bbs/                 # 聊天室 CGI 與紀錄
├── data/                # 遊戲靜態／半動態資料：海域、港口、商品、船艦、任務
├── city/                # 玩家建立的港町資料
├── usrdir/              # 玩家帳號資料（執行時產生；本倉庫只保留 index.html）
└── img/                 # 船艦、地圖、人物與 UI 圖像
```

其他說明檔如 `manual.htm`、`how.html`、`explanation.html`、`ReadMe.htm`、`readme.txt` 保留了原始玩法或安裝說明；本 `README.md` 聚焦於程式架構與從程式碼推得的玩法總覽。

## 執行入口與請求流程

### 1. 首頁與登入

- 使用者進入 `sea.cgi`。
- 若沒有 `mode` 參數，`sea.cgi` 載入 `rank.cgi`，顯示遊戲標題、建立帳戶連結、玩法說明連結、ID／密碼登入表單與排行榜。
- 登入送出 `mode=play` 後，主流程讀取玩家資料、設定 Cookie、載入所在地港口／海域資料，輸出玩家主畫面。

### 2. 模式分派

`sea.cgi` 是登入後的主要控制器。它依 `mode` 參數動態 `require` 對應模組，再呼叫同名子程序：

| mode 類型 | 載入檔案 | 功能 |
| --- | --- | --- |
| `play` | `sea.cgi` | 顯示主畫面、狀態、地圖與目前可執行行動 |
| `trade_buy` / `trade_sell` | `trade.cgi` | 交易品購買／賣出 |
| `move` | `move.cgi` | 出航到港口或其他海域 |
| `bar_trade` / `adven` | `bar.cgi` | 酒場補給、雇用、冒險情報 |
| `buy_ship` / `sell_ship` / `rep_ship` | `yard.cgi` | 造船所買賣修船 |
| `battle` | `battle.cgi` | 玩家間艦隊戰鬥 |
| `city*` | `city.cgi` | 城鎮主流程與建城 |
| `ctrade*` | `ctrade.cgi` | 城鎮市場、銀行、購買城鎮資產 |
| `cadmin*` | `cadmin.cgi` | 城鎮擁有者管理 |
| `cbattle_atk` | `cbattle.cgi` | 攻擊城鎮 |

每個會改變狀態的表單通常都帶有隱藏欄位：

- `id`：玩家 ID
- `ps`：密碼
- `mode`：要執行的動作
- `reload`：防止重複送出的動作 token

### 3. 主畫面組成

登入後的 `play` 畫面大致分成三欄：

1. **左欄**：世界地圖指標與艦隊排列圖。
2. **中欄**：玩家狀態、貨倉、資金、水手、等級、能力、名聲、財寶、訊息，以及港內移動／行動選單。
3. **右欄**：依目前位置與行動點顯示可操作功能，例如移動、交易、造船、酒場、城鎮、襲擊或航行倒數。

## 資料與持久化設計

本遊戲沒有資料庫，所有狀態皆存在純文字檔中。

### 檔案鎖定

`Nfile.pm` 包裝讀寫流程，建立全域鎖檔 `lock_Nfile`，用 `flock`，若不可用則退回 symlink／mkdir 鎖。它支援：

- `new Nfile($file, 'read')`：唯讀。
- `new Nfile($file, 'save')`：讀取後可寫回，物件解構時可自動保存。
- `new Nfile($file, 'new')`：建立新檔。

因為鎖是全域式的，同一個請求中開啟多個 `Nfile` 物件時會維持鎖定，直到所有物件關閉或寫回。這可降低多玩家同時操作造成資料檔互相覆寫的風險。

### 玩家資料

玩家資料存於 `usrdir/<id>.dat`。第一行是玩家主狀態，使用 `<>` 分隔欄位；後續行是履歷。主要欄位包括：

- ID、名稱、密碼、性別。
- 資金、戰鬥力、指揮力、航海力。
- 冒險／海賊／商人名聲。
- 食物、水手、貨物、船艦。
- 所在海域、港口、港內位置。
- 戰術、履歷、任務、好友、戰鬥限制。
- 最後操作時間、移動完成時間。
- 自我介紹、郵件、網址、Host／OS／IP、瀏覽器。
- 財寶、擁有城鎮、預備欄、三種經驗值。

`sys.cgi` 的 `get_me` / `set_me` 負責登入玩家資料讀寫；`get_u` / `set_u` 負責其他玩家資料讀寫。

### 海域與港口資料

`data/1001.dat` 到 `data/1009.dat` 代表九個海域。每個檔案第一行是海域本身，後續每行是一個港口：

```text
港口編號<>港口名稱<>座標<>交易品清單<>物價倍率<>
```

交易品清單使用 `△` 分隔，每項為：

```text
商品名稱,基礎價格
```

物價倍率會因玩家買賣而升降，也會在玩家抵達港口時逐漸回歸接近 1.0。

### 商品、船艦與任務資料

- `data/base.dat`：交易品基準價格，供賣出非本港商品或查價使用。
- `data/item.dat`：船艦與能力提升品資料，格式大致為 `名稱<>類型<>圖片<>販售海域<>販售港口<>容量或加成<>耐久<>速度<>價格`。
- `data/未翻譯任務/a_list.dat`：酒場可購買的冒險情報清單。
- `data/未翻譯任務/*.dat`：任務步驟。每行描述下一個目的地、提示文字與可能取得的財寶。

### 城鎮資料

玩家建立的港町資料存於 `city/<port>.dat`，欄位包含：

- 擁有者 ID／名稱。
- 城鎮名稱、HP、資金。
- 城鎮寄售貨物、船艦、財寶。
- 介紹文字、稅率、出售狀態與指定買方。

`csys.cgi` 提供 `get_city` / `set_city`；城鎮功能再由 `city.cgi`、`ctrade.cgi`、`cadmin.cgi`、`cbattle.cgi` 分工處理。

## 主要程式模組說明

### `setting.cgi`：全域設定

包含遊戲標題、資料目錄、圖片目錄、表單方法、Cookie 名稱、密碼加密開關、IP 限制、未更新刪除天數、網站連結、CSS 與各 CGI 路徑。

部署時最重要的是修改：

- `$adps`：管理者密碼，預設是 `0123`，正式使用必須更改。
- `$usrdir`、`$datadir`、`$citydir`：資料目錄。
- `$hom_url`、`$hom_lbl`：所屬網站資訊。
- `$def_cp`：是否啟用 `crypt` 密碼雜湊。

### `sea.cgi`：遊戲核心

此檔案同時是主入口與核心 UI 組裝者。它負責：

- 驗證玩家 ID 格式與密碼。
- 依 `mode` 分派動作。
- 顯示世界地圖、艦隊、玩家狀態與目前可用功能。
- 計算船隊總容量、速度、剩餘貨艙。
- 檢查任務進度、戰術狀態、沉沒與航行中倒數。
- 提供港內位置切換：港口、交易、造船、酒場、城鎮、襲擊。

### `new.cgi`：帳號建立

玩家建立帳號時需輸入船長名、密碼、Email／網址、性別、出發地點，並分配戰鬥力、指揮力、航海力。初始能力總和由 `$capacity` 決定，初始資金由 `$def_fm` 決定。系統會檢查名稱、Email、URL、IP／瀏覽器資訊是否重複，並產生遞增的五位數 ID。

### `trade.cgi`：交易系統

港口交易是商人路線核心：

- 購買時檢查資金與剩餘貨艙。
- 賣出時從玩家貨物扣除數量並增加資金。
- 商品成交價 = 商品基礎價 × 港口物價倍率 × 商品週期波動。
- 買入會推高港口物價，賣出會壓低物價。
- 商人經驗可能觸發議價折扣。
- 交易金額會增加商人名聲與商人經驗。

### `move.cgi`：航行系統

玩家可從目前港口出航到同海域其他港，或移動到其他海域。航行時間依兩地座標距離、`$time_scale` 與船隊速度計算。若從海上／跨海域移動，會依航行時間與水手數消耗食物；食物不足則無法出航。航行期間主畫面顯示船隻動畫與倒數，抵達後可再行動。

### `yard.cgi`：造船所

造船所提供：

- 購買船艦。
- 賣出船艦。
- 修理船艦。
- 購買能力提升品，提升戰鬥力、指揮力或航海力。

船艦資料來自 `data/item.dat`，不同船艦有不同容量、耐久、速度、價格與販售地點。能力提升品價格會隨玩家能力總和指數上升。

### `bar.cgi`：酒場與冒險

酒場功能包含：

- 雇用／解雇水手。
- 購買／丟棄食物。
- 查看同港其他玩家，並互相加入好友紀錄。
- 購買冒險情報，開啟任務線。

冒險情報會指定任務資料檔。玩家依序抵達任務要求的海域、港口與港內位置後，會推進任務、增加冒險名聲／經驗，並可能取得財寶。

### `battle.cgi`：玩家戰鬥

玩家可在同一地點襲擊其他玩家。戰鬥重點：

- 需在相同港口、海上位置或海域條件下才可攻擊。
- 有連續戰鬥與同一對手限制。
- 戰術分為好戰、適度、回避；小艦隊可投降，港口可停靠。
- 戰鬥會依玩家戰鬥力、指揮力、船艦耐久與戰術計算攻擊與迴避。
- 戰勝可奪取資金、貨物或財寶，並改變海賊／冒險／商人名聲與經驗。
- 船隻耐久歸零會沉沒；若在海上全滅則遊戲結束並刪除帳號。

### 城鎮系統

城鎮讓玩家從單純航海擴展到據點經營：

- `city.cgi`：若港口無城鎮，可花費資金建立；若已有城鎮，顯示城鎮入口。
- `cadmin.cgi`：城鎮擁有者可提領資金、修復 HP、搬入貨物／船艦／財寶上架、設定介紹、銀行與出售條件。
- `ctrade.cgi`：其他玩家可購買城鎮庫存、使用城鎮銀行、購買出售中的城鎮。
- `cbattle.cgi`：玩家可用武力攻擊或付費破壞工作降低城鎮 HP；HP 歸零時城鎮資料會被刪除，攻擊者海賊名聲上升、商人名聲下降。

### 社交與輔助功能

- `mail.cgi`：玩家間寄信。
- `list.cgi`：玩家列表、ID 列表、玩家 Profile、船艦列表、個人履歷與事件紀錄。
- `intro.cgi`：自我介紹修改。
- `chat.cgi` + `bbs/say.cgi`：主畫面內嵌聊天室。
- `rank.cgi`：依資金、冒險、海賊、商人等指標排名，首頁顯示前三名與新人榜。
- `event.cgi`：記錄全服事件，例如城鎮陷落。

## 遊戲玩法總覽

### 1. 建立船長

玩家在首頁點選「創立帳戶」，輸入船長名稱與密碼，選擇性別與出發港，並分配三種能力：

- **戰鬥力**：影響玩家戰鬥與攻城的攻擊力。
- **指揮力**：影響迴避率與戰鬥表現。
- **航海力**：影響船隊速度，進而縮短航行時間。

建立成功後取得玩家 ID，登入時需使用 ID 與密碼。

### 2. 擴張艦隊與補給

初期玩家需要到造船所購買船隻。船隊決定：

- **貨倉容量**：能裝多少交易品、食物與水手。
- **耐久 HP**：戰鬥或攻城時承受傷害。
- **速度**：航行耗時。

酒場可雇用水手與購買食物。水手會佔容量並消耗食物，但對戰鬥與部分行動有幫助；食物不足會限制遠航。

### 3. 貿易賺錢

玩家在港口交易所購買當地商品，航行到其他港口後賣出。獲利關鍵是：

- 低價港買入、高價港賣出。
- 注意港口物價倍率。
- 商品有週期性波動。
- 船隊容量越大，一趟能帶的貨越多。
- 商人經驗提高後可能取得議價優勢。

交易能提升商人名聲與商人經驗，是穩定累積資金的主要方式。

### 4. 航行探索

在港口的「港口」行動可選擇目的地。移動時間由距離與速度決定。遠航需要準備足夠食物；某些財寶可降低食物消耗。航行結束後，玩家可在目的港進行交易、造船、酒場、城鎮或戰鬥行動。

### 5. 冒險任務與財寶

在酒場購買冒險情報後，玩家會得到提示。任務通常要求依序前往指定海域、港口與港內位置。完成整條任務線後：

- 增加冒險名聲。
- 增加冒險經驗。
- 可能獲得財寶。

財寶除了收藏，也可能具備特殊效果，例如戰鬥防護、增加攻擊回合、避免戰鬥、降低食物消耗或保護財寶。

### 6. 海賊與戰鬥

玩家可在「襲撃」行動中攻擊附近玩家。戰鬥帶來高風險高報酬：

- 勝利可能奪取資金、貨物或財寶。
- 會提升海賊相關名聲／經驗。
- 可能造成船隻沉沒與資源損失。
- 對方採取回避、投降或有特殊財寶時，可能減少或避免損害。
- 遊戲限制短時間連續攻擊，避免同一對手被過度追擊。

### 7. 城鎮經營與攻防

當港口沒有玩家城鎮時，資金足夠的玩家可建立港町。城鎮主可：

- 儲存或提領城鎮資金。
- 修復城鎮 HP。
- 搬入貨物、船艦、財寶並設定價格。
- 提供城鎮銀行服務。
- 修改城鎮介紹或出售城鎮。

其他玩家可購買城鎮商品，也可攻擊城鎮。城鎮 HP 歸零時會毀滅。

### 8. 排行與長期目標

遊戲沒有固定終局，長期目標包括：

- 成為資金最多的大富豪。
- 完成冒險任務並收集財寶。
- 成為海賊王，透過戰鬥建立威名。
- 經營或爭奪重要港町。
- 擴張高容量、高速度、高耐久的艦隊。
- 在排行榜中取得冒險、海賊、商人或資金排名。

## 部署與維護注意事項

1. **需要可執行 Perl CGI 的 Web 伺服器**：`.cgi` 檔案第一行是 `#!/usr/bin/perl`，部署環境需符合或調整 shebang。
2. **資料檔需可寫入**：`usrdir/`、`city/`、`data/*.dat`、`event.dat`、`bbs/log.dat` 等在遊戲運作中可能被寫入。
3. **務必修改管理密碼**：`setting.cgi` 的 `$adps = '0123'` 只是預設值，不適合公開部署。
4. **確認密碼策略**：預設 `$def_cp = 0`，代表密碼不加密；正式環境建議啟用並評估相容性。
5. **檔案鎖為全域鎖**：高流量時所有資料寫入會互相等待，這是此類純文字 CGI 遊戲的架構限制。
6. **輸入與 HTML 安全性需額外審視**：本程式歷史悠久，許多輸出直接插入 HTML；若要公開服務，建議先做完整安全稽核。
7. **任務資料夾名稱**：目前冒險資料在 `data/未翻譯任務/`，但 `sea.cgi` 預設 `$adfiles = 'a_list.dat'`，若實際部署要啟用這些任務，需確認檔案位置與設定一致。

## 開發備註

- 程式大量使用全域變數，模組透過 `require` 共享同一個執行環境。
- 資料格式主要靠固定欄位順序與分隔字元維持，修改欄位時需同步更新所有 `split(/<>/)` 與 `join('<>')` 的位置。
- 多數表單動作都會呼叫 `get_me` 重新讀取玩家、檢查 `reload` token，最後 `play` 再統一顯示畫面與 `set_me` 寫回。
- 若要新增功能，建議沿用既有模式：在 `sea.cgi` 增加 mode 分派、建立子程序、使用 `id_ps` 與 `reload` 保留登入與防重送欄位，並以 `Nfile` 讀寫資料。
